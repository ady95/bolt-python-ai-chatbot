# 이벤트 처리 수정 사항

## 배경

기존 봇은 채널에 초대된 경우 `@{app_name}` 멘션이 있어야만 응답했습니다. 다음 두 가지 동작을 추가하기 위해 메시지 이벤트 핸들러를 수정했습니다.

1. 멘션 없이도 채널 메시지에 thread(댓글) 형태로 응답
2. thread 안의 메시지에 대해서는 같은 thread에서 연속해서 응답

## 수정 파일

- `listeners/events/app_messaged.py`

## 주요 변경 내용

### 1. 무한 루프 / 노이즈 방지

```python
if event.get("bot_id") or event.get("subtype") is not None:
    return
```

- 봇 자신이 보낸 메시지(`bot_id` 존재)는 무시
- `subtype`이 있는 이벤트(메시지 수정/삭제, 채널 입장 알림 등)도 무시
- 추가로 `user_id == bot_user_id`도 한 번 더 필터링

### 2. 채널 멘션 중복 방지

```python
if not is_dm and bot_user_id and f"<@{bot_user_id}>" in text:
    return
```

- 채널/그룹 메시지에 `<@bot_user_id>`가 포함되어 있으면 `app_messaged` 핸들러는 무시
- 멘션은 기존 `app_mentioned` 핸들러가 처리하도록 위임 → 한 메시지에 두 번 응답하는 문제 방지
- `bot_user_id`는 Bolt가 자동 주입하는 `context: BoltContext`에서 가져옴

### 3. 응답 위치 결정 로직

```python
if is_dm:
    reply_thread_ts = thread_ts
else:
    reply_thread_ts = thread_ts or event.get("ts")
```

| 상황 | 응답 위치 |
|------|-----------|
| DM, 일반 메시지 | 채널 본문 |
| DM, thread 안의 메시지 | 같은 thread |
| 채널/그룹, 일반 메시지 | **그 메시지를 root로 새 thread 생성** |
| 채널/그룹, thread 안의 메시지 | **같은 thread에 연속 응답** |

- 채널/그룹/MPIM에서는 항상 thread로 응답
- 이미 thread 안에 있으면 해당 `thread_ts` 사용 → 요구사항 2 충족
- thread 밖이라면 현재 메시지의 `ts`를 thread root로 사용 → 요구사항 1 충족

### 4. 시스템 프롬프트 분기

```python
if is_dm:
    response = get_provider_response(
        user_id, text, conversation_context, DM_SYSTEM_CONTENT
    )
else:
    response = get_provider_response(user_id, text, conversation_context)
```

- DM은 기존대로 `DM_SYSTEM_CONTENT` 사용
- 채널은 기본 시스템 프롬프트 사용 (`app_mentioned` 핸들러와 일관성 유지)

### 5. 에러 처리 안정화

기존 코드는 `say()` 호출 전에 예외가 발생하면 `waiting_message`가 정의되지 않아 except 블록에서 또 다른 `NameError`가 발생했습니다.

```python
waiting_message = None
try:
    ...
    waiting_message = say(text=DEFAULT_LOADING_TEXT, thread_ts=reply_thread_ts)
    ...
except Exception as e:
    logger.error(e)
    if waiting_message:
        client.chat_update(...)
```

- `waiting_message = None`으로 초기화
- except 블록에서 `if waiting_message:` 가드 추가

## 전체 수정 코드

```python
from ai.ai_constants import DM_SYSTEM_CONTENT
from ai.providers import get_provider_response
from logging import Logger
from slack_bolt import BoltContext, Say
from slack_sdk import WebClient
from ..listener_utils.listener_constants import DEFAULT_LOADING_TEXT
from ..listener_utils.parse_conversation import parse_conversation

"""
Handles message events for the bot. Responds in DMs, and in channels/groups where the bot
is a member it replies in a thread to every user message. Threaded replies continue the
conversation in the same thread.
"""


def app_messaged_callback(
    client: WebClient,
    event: dict,
    logger: Logger,
    say: Say,
    context: BoltContext,
):
    # Ignore bot messages and message subtypes (edits, joins, deletes, etc.)
    # to prevent infinite loops and noise.
    if event.get("bot_id") or event.get("subtype") is not None:
        return

    channel_id = event.get("channel")
    thread_ts = event.get("thread_ts")
    user_id = event.get("user")
    text = event.get("text") or ""
    channel_type = event.get("channel_type")
    bot_user_id = context.bot_user_id

    # Skip the bot's own user messages just in case.
    if user_id and bot_user_id and user_id == bot_user_id:
        return

    is_dm = channel_type == "im"

    # In channels/groups, if the bot is @mentioned, let app_mention handler
    # process it to avoid duplicate responses.
    if not is_dm and bot_user_id and f"<@{bot_user_id}>" in text:
        return

    # Decide which thread_ts to reply in:
    # - DMs: keep current behavior (only thread if user already in a thread)
    # - Channels: always reply in thread (use existing thread or start one on this message)
    if is_dm:
        reply_thread_ts = thread_ts
    else:
        reply_thread_ts = thread_ts or event.get("ts")

    waiting_message = None
    try:
        # Build conversation context from the thread when applicable.
        conversation_context = ""
        if thread_ts:
            conversation = client.conversations_replies(
                channel=channel_id, limit=10, ts=thread_ts
            )["messages"]
            conversation_context = parse_conversation(conversation[:-1])

        waiting_message = say(text=DEFAULT_LOADING_TEXT, thread_ts=reply_thread_ts)

        if is_dm:
            response = get_provider_response(
                user_id, text, conversation_context, DM_SYSTEM_CONTENT
            )
        else:
            response = get_provider_response(user_id, text, conversation_context)

        client.chat_update(
            channel=channel_id, ts=waiting_message["ts"], text=response
        )
    except Exception as e:
        logger.error(e)
        if waiting_message:
            client.chat_update(
                channel=channel_id,
                ts=waiting_message["ts"],
                text=f"Received an error from Bolty:\n{e}",
            )
```

## 주의 사항

- 봇이 초대된 채널의 **모든 사용자 메시지**에 응답하게 됩니다.
  트래픽이 많은 채널에서는 매우 시끄러워질 수 있고 토큰 사용량도 급증할 수 있으니,
  필요한 경우 키워드/명령어 등으로 한 번 더 필터를 거는 것을 권장합니다.
- 변경 사항 반영을 위해 `app.py`를 재시작해야 합니다.

  ```bash
  python app.py
  ```
